# Skill: GCP Cloud Console Navigator

This skill provides instructions for configuring Google Cloud Platform (GCP) services specifically for the AI Voice Receptionist and Gemini Enterprise CX Agent Studio.

## 🛠️ Service Account Provisioning
When creating a service account for the Receptionist:
1. **Name**: `ariel-receptionist-sa`
2. **Roles**:
    - `roles/aiplatform.user` (Agent Platform access)
    - `roles/discoveryengine.user` (Vertex AI Search / Data Store access)
    - `roles/sheets.editor` (Google Sheets write access)
3. **Key Management**: Use the Google Cloud Console to generate a JSON key and store it securely (e.g., in a `.env` file or Secret Manager).

## 🧩 CX Agent Studio Configuration
Use the following steps to configure the backend:
1. **Agents Page**: Go to the "Agents" page in the GCP Console.
2. **Grounding**:
    - Enable **Google Search** for general knowledge.
    - Connect **Vertex AI Search** for proprietary knowledge.
3. **MCP Connectivity**:
    - If using an MCP server, ensure it is reachable via a public URL or connected via VPC.
    - Register the server in the "Tools" section of Agent Studio.

## 📡 Webhook / Edge Function Setup
1. **Endpoint**: Use a Cloud Run or Supabase Edge Function as the webhook.
2. **Authentication**: Ensure the webhook verifies the JWT from CX Studio if required.
3. **Regionality**: Match the region of the CX Studio agent (e.g., `us-central1`) with the webhook deployment to minimize latency.

## ⚡ CLI Automation
The `agents-cli` can automate several of the manual steps above:

- **Infra Provisioning**: `agents-cli infra single-project` sets up basic GCP resources via Terraform.
- **Project Info**: `agents-cli info` provides the current Project ID, Region, and Deployment Target.
- **Login**: `agents-cli login --interactive` handles the OAuth flow for your terminal.

## 🛡️ Best Practices
- **Least Privilege**: Only grant the specific roles listed above.
- **Quota Monitoring**: Monitor API usage in the GCP "APIs & Services" dashboard to avoid rate limiting.
- **Logging**: Enable "Audit Logs" in GCP to track agent actions.
