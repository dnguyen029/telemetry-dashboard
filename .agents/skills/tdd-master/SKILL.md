# 🧪 TDD Master Skill

Unified authority for the Red-Green-Refactor cycle.

## 🔴 1. RED Phase

- Write a failing test that defines the desired behavior.
- Ensure the test fails for the expected reason.

## 🟢 2. GREEN Phase

- Write the MINIMAL amount of code to make the test pass.
- Do not optimize or add features yet.

## 🔵 3. REFACTOR Phase

- Clean up the code while keeping the tests green.
- Remove duplication and improve readability.

## 🛡️ Best Practices

- **Isolation**: Tests should be independent and side-effect free.
- **Coverage**: Focus on edge cases and error boundaries.
- **Speed**: TDD tests should be fast to encourage frequent execution.
